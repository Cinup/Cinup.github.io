Java日志总结

发表于 2020-04-06 | 分类于 java

简介

最近在项目方案调研过程中，发现Log4j可以将日志发送到远程服务器，但是Log4j2似乎又不行，感觉自己对Java日志框架体系很陌生，因此来梳理一下。

日志

日志的作用

日志是程序员在编程的时候主动输出的记录，主要用于记录程序运行的过程，方便排错。在项目出现问题时，第一步都是先阅读项目的日志，通过日志来大致定位问题，譬如有没有收到数据，有没有异常信息等。

日志框架

JUL

Java自带日志框架，在java.util.logging包下。由于其API不够完善，对开发者不够灵活，使用的比较少。

Log4j

Log4j是Apache的一个开源项目，通过使用Log4j，可控制日志信息输送的目的地是控制台、文件、数据库等。也可以控制每一条日志的输出格式，通过定义每一条日志信息的级别，能够更加细致的控制日志的生成过程。

引入依赖

Log4j主要由三个组件构成

  • logger负责日志信息的输出
  • appender负责日志信息的输出目的地
  • layout负责日志信息的输出格式

配置文件log4j.properties

Log4j2

Slf4j

slf4j(The Simple Logging Facade for Java)并不是日志框架，而是作为一个简单的门面或抽象，用来服务于各种各样的日志框架，比如java.util.logging、logback和log4j。它使用标准的接口输出日志，允许自由更换日志框架而无需修改代码。
