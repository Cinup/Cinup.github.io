集合类框架

首先先从接口方面看一下集合类等框架。

图1

Collection接口是所有集合的父接口，它继承了Iterable接口。Iterable接口的作用是返回一个的迭代器对象。Iterator接口只有三个方法。

集合类主要包含两种容器，一种是集合，存储一个元素，一种是图，存储键/值对。

Collection接口有三个子接口:List，Set，Queue。这个三个接口各有各的特点，使用场景也不同。

接口 特点
List 存储一组可重复，有序（插入顺序）的对象
Set 存储不重复，无序的对象，最多允许有一个null元素对象
Queue 队列，存储对象先进先出。

Map则是依赖Collection接口的，values()方法会返回一个Collection类型的对象。
