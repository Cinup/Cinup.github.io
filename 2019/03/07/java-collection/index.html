发表于 2019-03-07 | 更新于 2019-03-08 | 分类于 java

简介

Java集合类解析。

集合类框架

首先先从接口方面看一下集合类等框架。

图1

Collection接口是所有集合的父接口，它继承了Iterable接口。Iterable接口的作用是返回一个的迭代器对象。Iterator接口只有三个方法，用于元素的遍历。

集合类主要包含两种容器，一种是集合，存储一个元素，一种是图，存储键/值对。

Collection接口有三个子接口:List，Set，Queue。这个三个接口各有各的特点，使用场景也不同。Map则是依赖Collection接口的，values()方法会返回一个Collection类型的对象。

接口 特点
List 存储一组可重复，有序(插入顺序)的对象
Set 存储不重复，无序的对象，最多允许有一个null元素对象
Queue 队列，存储对象先进先出
Map 键值对，键必须唯一

ArrayList

ArrayList继承自AbstractList的List实现类，它是一个可以动态变化的索引序列，内部存储是Object数组。

可以通过initialCapacity参数指定ArrayList初始容量的大小，如果不指定大小那么会使用默认大小。

虽然默认大小是10，但是构造函数并不会立即分配Object数组空间，而是让内部数组引用指向DEFAULTCAPACITY_EMPTY_ELEMENTDATA，以此来标记它是初始容量的ArrayList，等到真正向ArrayList中添加数据时，才开始分配空间。

扩容

ArrayList能够不断添加数据自动实现扩容，其实质是底层的数据拷贝。当插入的数据，如果发现容量已经满了，就会开始扩容，扩容后的容量大概等于扩容前的1.5倍。

总结

ArrayList的实质是对象数组，因此add和remove数据会频繁的拷贝数组，非常影响性能。总的来说，ArrayList的优点是随机访问，缺点是数据插入和删除数据是可能会发生扩容造成性能下降。

LinkedList

LinkedList同时实现了List接口和Deque接口，是一种可以在任意位置进行高效数据增删操作的有序序列。LinkedList继承于AbstractSequentialList，因此它是双向链表，同时又可以当做栈，队列或双端队列使用。

内部存储使用静态内部类。

总结

因此LinkedList内部使用Node静态内部类存储数据，Node除了存储数据外，还有两个Node引用，一个指向前一个Node，一个指向后一个的Node，像链条一样一环扣一环，因此并不需要连续的空间来存储整个LinkedList，数据的插入和删除只需要修改对应Node中prev和next的引用即可，不需要重新开辟内存拷贝过去，因此比ArrayList更加高效。但是访问任意一个Node都必须遍历它前(后)边的所有Node，所以查询效率不如ArrayList高。

Vector

Vector和ArrayList大体上是差不多的，有一些区别。首先，ArrayList在多线程是不安全的，而Vector的操作的数据方法都使用synchronized修饰，是线程安全的。其次，Vector的初始容量默认是10，可以指定扩容大小，每次扩容在原始大小上增加扩容大小。如果不指定扩容大小，默认则扩容一倍。

Map

HashMap

HashMap是散列表，是数组和链表的结合体。其本质就是把链表分组，将具有相同特点的数组放在一个链表里，然后再通过数组将链表组织起来，这样就可以有效地缩小搜索链表的范围。

而这个相同特点就是hash值，因此HashMap需要键的唯一性。

HashMap内部存储数据的形式是链表数组。

构造函数

因为HashMap使用数组table来组织数据，使用数组就需要考虑数组大小和扩容问题。HashMap有4个构造函数:

当不指定初始大小时，HashMap的table容量大小为16。当指定初始容量时，HashMap的数组容量为第一个不小于初始值的2的N次幂，例如当initialCapacity为7时，初始容量为8。

HashMap除了初始大小外，还有一个叫做容载因子loadFactor的变量，它的初始值为0.75。

当table中的链表数/table的大小 > 容载因子时，说明table快满了，这时再插入数据有很大可能会发生冲突，因此需要table需要扩容了。容载因子就是控制扩容时机的。

LinkedHashMap

LinkedHashMap继承自HashMap，HashMap因为存储结构的特点，它的元素迭代顺序和插入顺序会完全不一样，LinkedHashMap加强了这个方面，能够使得元素迭代顺序和插入顺序一致。

LinkedHashMap内部存储数据还是数组链表，只是再额外增加一个Entry链表。这样只需要在插入或删除数据时维护好Entry链表，通过头指针遍历顺序就是插入顺序。LinkedHashMap存储的数据是HashMap加链表，通过增在额外的存储空间来保证迭代的顺序。
