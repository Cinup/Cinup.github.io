Java集合类

发表于 2019-03-07 | 更新于 2020-11-19 | 分类于 java

简介

Java集合类知识总结。

集合类框架

首先先从接口方面看一下集合类的框架。

图1

Collection接口是所有集合的父接口，它继承了Iterable接口。Iterable接口的作用是返回一个的迭代器Iterator对象。

集合类主要包含两种容器，一种是集合，存储一个元素，一种是图，存储一对键值对。

Collection接口有三个子接口:List，Set，Queue。这个三个接口各有各的特点，使用场景也不同。Map则是依赖Collection接口的，values()方法会返回一个Collection类型的对象。

接口 特点
List 存储一组可重复，有序(插入顺序)的对象
Set 存储不重复，无序的对象，最多允许有一个null元素对象
Queue 队列，存储对象先进先出
Map 键值对，键必须唯一，值可以重复

List

ArrayList


ArrayList是继承自AbstractList的List实现类，它是一个可以动态变化的索引序列，内部存储是Object数组。

transient Object[] elementData;

构造函数

ArrayList有3个构造函数:

public ArrayList() {}
public ArrayList(int initialCapacity) {}
public ArrayList(Collection<? extends E> c) {}

可以通过initialCapacity参数指定ArrayList初始容量的大小，如果不指定大小那么会使用默认大小。

private static final int DEFAULT_CAPACITY = 10;

虽然默认大小是10，但是构造函数并不会立即分配Object数组空间，而是让内部数组引用指向DEFAULTCAPACITY_EMPTY_ELEMENTDATA，以此来标记它是初始容量的ArrayList，等到真正向ArrayList中添加数据时，才开始分配空间。

private static final Object[] DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {};
//无参构造函数
public ArrayList() {
    this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
}

扩容

ArrayList能够不断添加数据自动实现扩容，其实质是底层的数据拷贝。当插入的数据，如果发现容量已经满了，就会开始扩容，扩容后的容量大约等于扩容前的1.5倍。

public boolean add(E e) {
    modCount++;
    //两个public add方法内部都是调用private方法
    add(e, elementData, size);
    return true;
}
private void add(E e, Object[] elementData, int s) {
    if (s == elementData.length)
        //扩容
        elementData = grow();
    //在末尾添加数据
    elementData[s] = e;
    size = s + 1;
}
//grow
private Object[] grow() {
    return grow(size + 1);
}
//
private Object[] grow(int minCapacity) {
    //利用Arrays.copyOf方法扩容并拷贝数组
    return elementData = Arrays.copyOf(elementData,
                                       newCapacity(minCapacity));
}
//计算新的容量
private int newCapacity(int minCapacity) {
    int oldCapacity = elementData.length;
    // >>1 右移一位，相当于除以2，因此扩容扩1.5倍
    int newCapacity = oldCapacity + (oldCapacity >> 1);
    if (newCapacity - minCapacity <= 0) {
        if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA)
            return Math.max(DEFAULT_CAPACITY, minCapacity);
        if (minCapacity < 0) // overflow
            throw new OutOfMemoryError();
        return minCapacity;
    }
    return (newCapacity - MAX_ARRAY_SIZE <= 0)
        ? newCapacity
        : hugeCapacity(minCapacity);
}

总结

ArrayList的实质是对象数组，因此add和remove数据会频繁的拷贝数组，非常影响性能。总的来说，ArrayList的优点是随机访问，缺点是数据插入和删除数据是可能会发生扩容造成性能下降。

LinkedList


LinkedList同时实现了List接口和Deque接口，是一种可以在任意位置进行高效数据增删操作的有序序列。LinkedList继承于AbstractSequentialList，因此它是双向链表，同时又可以当做栈，队列或双端队列使用。

内部存储使用静态内部类。

private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;
    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}
//头结点
transient Node<E> first;
//尾结点
transient Node<E> last;

构造函数

LinkedList只有2个构造函数。

public LinkedList() {}
public LinkedList(Collection<? extends E> c) {}

总结

因此LinkedList内部使用Node静态内部类存储数据，Node除了存储数据外，还有两个Node引用，一个指向前一个Node，一个指向后一个的Node，像链条一样一环扣一环，因此并不需要连续的空间来存储所有元素，数据的插入和删除只需要修改对应Node中prev和next的引用即可，不需要重新开辟内存拷贝过去，因此比ArrayList更加高效。但是访问任意一个Node都必须遍历它前(后)边的所有Node，所以查询效率不如ArrayList高。

Vector


Vector和ArrayList使用方法上大体上是差不多的，不过有一些区别。首先，ArrayList在多线程是不安全的，而Vector的操作的数据方法都使用synchronized修饰，是线程安全的。其次，Vector的初始容量默认是10，可以指定扩容大小，每次扩容在原始大小上增加扩容大小。如果不指定扩容大小，默认则扩容一倍。

构造函数

//默认容量10
public Vector() {}
//指定初始容量
public Vector(int initialCapacity) {}
//指定初始容量和扩容大小
public Vector(int initialCapacity, int capacityIncrement) {}
public Vector(Collection<? extends E> c) {}

总结

Vector就是多线程下使用的ArrayList，同时Vector的扩容机制可以定制，可以根据需要自由选择，ArrayList只能选择初始容量。

Stack


Stack继承自Vector，能够以栈式(先进后出)访问数组，也是线程安全的。

构造函数

public Stack() {}

List总结

类 访问方式 线程安全 特点
ArrayList 随机访问 否 访问快，操作慢

Vector 随机访问 是 访问快，操作慢

Map

HashMap


HashMap是散列表，是数组和链表的结合体。其本质就是把链表分组，将具有相同特点的数组放在一个链表里，然后再通过数组将链表组织起来，这样就可以有效地缩小搜索链表的范围。


而这个相同特点就是键的hash值，因此HashMap需要键的唯一性。

HashMap内部存储数据的形式是链表数组。

//数组
transient Node<K,V>[] table;
//静态内部类Node
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;
}

构造函数

因为HashMap使用数组table来组织数据，使用数组就需要考虑数组大小和扩容问题。HashMap有4个构造函数:

public HashMap() {}
public HashMap(int initialCapacity) {}
public HashMap(int initialCapacity, float loadFactor) {}
public HashMap(Map<? extends K, ? extends V> m) {}

当不指定初始大小时，HashMap的table容量大小为16。当指定初始容量时，HashMap的数组容量为第一个不小于初始值的2n，例如当initialCapacity为7时，初始容量为8。

HashMap除了初始大小外，还有一个叫做负载因子loadFactor的变量，它的初始值为0.75。

//初始容量
static final int DEFAULT_INITIAL_CAPACITY = 1 << 4
//初始负载因子
static final float DEFAULT_LOAD_FACTOR = 0.75f;

当table中的元素 > 负载因子*table的大小时，说明table快满了，这时再插入数据有很大可能会发生冲突，因此需要table需要扩容了。负载因子就是控制扩容时机的。

在java8中，如果插入数据一直发生冲突，这个时候链表可能会变成红黑树，下面是转换成红黑树的必要条件。

//转换为红黑树时的链表长度
static final int TREEIFY_THRESHOLD = 8;
//退化为链表时的红黑树长度
static final int UNTREEIFY_THRESHOLD = 6;
//转换为红黑树所需的最小键值对数目
static final int MIN_TREEIFY_CAPACITY = 64;

Hash

HashMap的hash计算函数如下，计算出hash值再对数组大小取余数，就可以得到Node链表在table中的位置，再访问table就可以得到Node链表。值为null的对象存在table[0]中。

static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

LinkedHashMap


LinkedHashMap继承自HashMap，HashMap因为存储结构的特点，它的元素迭代顺序和插入顺序会完全不一样，LinkedHashMap加强了这个方面，能够使得元素迭代顺序和插入顺序一致。

LinkedHashMap内部存储数据还是数组链表，只是再额外增加一个Entry链表。这样只需要在插入或删除数据时维护好Entry链表，通过头指针遍历Entry链表的顺序就是插入顺序。LinkedHashMap存储的数据是HashMap加链表，通过增在额外的存储空间来保证迭代的顺序。


Entry是双向链表，结构如下:

static class Entry<K,V> extends HashMap.Node<K,V> {
    Entry<K,V> before, after;
    Entry(int hash, K key, V value, Node<K,V> next) {
        super(hash, key, value, next);
    }
}

//头节点
transient LinkedHashMap.Entry<K,V> head;
//尾节点
transient LinkedHashMap.Entry<K,V> tail;

构造函数

LinkedHashMap有5个构造函数，其中4个和HashMap完全一样，还有一个构造函数用来确定迭代顺序。

public LinkedHashMap(int initialCapacity,
                     float loadFactor,
                     boolean accessOrder) {}

accessOrder用来确定迭代顺序，当其值为true时，迭代顺序是访问顺序，false时，迭代顺序是插入顺序，默认为false。

TreeMap


TreeMap是基于红黑树实现的，能够根据键进行排序。其结点结构如下:

static final class Entry<K,V> implements Map.Entry<K,V> {
    K key;
    V value;
    Entry<K,V> left;
    Entry<K,V> right;
    Entry<K,V> parent;
    boolean color = BLACK;
    //构造函数
    Entry(K key, V value, Entry<K,V> parent) {
        this.key = key;
        this.value = value;
        this.parent = parent;
    }
}

构造函数

TreeMap有4个构造函数，能够通过构造函数指定排序排序的方法，默认的排序方法是键的自然排序。

public TreeMap() {}
public TreeMap(Comparator<? super K> comparator) {}
public TreeMap(SortedMap<K, ? extends V> m) {}
public TreeMap(Map<? extends K, ? extends V> m) {}

排序

TreeMap内部是一颗红黑树，红黑树就是一种特殊的二叉查找树。只要理解了红黑树，就可以理解为什么TreeMap可以实现排序的了。

HashTable


HashTable和上面的Map类继承自AbstractMap不一样，它继承自Dictionary。

HashTable在功能上基本上和HashMap一样，但是有一些差别。

在多线程方面，HashTable是线程安全的，而HashMap是不安全的。

HashTable和HashMap的hash函数不一样。

//HashTable
int hash = key.hashCode();
int index = (hash & 0x7FFFFFFF) % tab.length;
//HashMap,因为分布在不同函数中，为了对比有修改
int hash;
(key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
index = (table.length - 1) & hash

HashTable的初始默认容量为11，HashMap的初始默认容量为16，HashTable每次扩容增加一倍再1，而HashMap每次扩容增加一倍，而且容量一定是2的幂。

Map总结

类 实现方式 线程安全 特点
HashMap 链表数组 否 迭代无序
LinkedHashMap 链表数组+链表 否 迭代有序
TreeMap 红黑树 否 迭代无序
HashTable 链表数组 是 迭代无序

问题

为什么HashMap允许HashMap允许键值为null，HashTable不允许键值为null

从源码上看

HashMap有一个专门的hash函数，在key为null时，会将hash值赋值为0

static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

HashTable没有这样函数，因此在key为null时会由于调用key.hashCode()抛出NullPointerException，value为null时主动抛出NullPointerException。

public synchronized V put(K key, V value) {
    if (value == null) {
        throw new NullPointerException();
    }

    Entry<?,?> tab[] = table;
    int hash = key.hashCode();//key=null时会抛出NullPointerException
}

Set

HashSet


HashSet也是散列表，但是存储的不是键值对对象，而是单个对象。

HashSet内部的实现其实是依赖HashMap的。

private transient HashMap<E,Object> map;

构造函数

HashSet有5个构造函数，构造函数内部实际都在构造HashMap。

public HashSet() {
    map = new HashMap<>();
}
public HashSet(int initialCapacity) {
    map = new HashMap<>(initialCapacity);
}
public HashSet(int initialCapacity, float loadFactor) {
    map = new HashMap<>(initialCapacity, loadFactor);
}
HashSet(int initialCapacity, float loadFactor, boolean dummy) {
    map = new LinkedHashMap<>(initialCapacity, loadFactor);
}
public HashSet(Collection<? extends E> c) {
    map = new HashMap<>(Math.max((int) (c.size()/.75f) + 1, 16));
    addAll(c);
}

原理

HashSet内部使用HashMap，添加的却不是键值对，是因为HashSet把添加值当做HashMap的键，使用一个空的Object对象当做HashMap的值，HashMap键的唯一性保证了HashSet元素不重复。

private static final Object PRESENT = new Object();
public boolean add(E e) {
    return map.put(e, PRESENT)==null;
}

TreeSet


TreeSet只是在HashSet的基础上实现了NavigableSet接口，因此TreeSet和TreeMap一样，实现了排序。

TreeSet在内部实现上依赖NavigableMap的实现类。

private transient NavigableMap<E,Object> m;

构造函数

TreeSet有4个构造函数，构造函数内部实际上也是在构造NavigableMap。默认使用TreeMap。

public TreeSet() {
    //调用下面的private构造函数
    this(new TreeMap<>());
}
public TreeSet(SortedSet<E> s) {}
public TreeSet(Collection<? extends E> c) {}
//自定义比较器
public TreeSet(Comparator<? super E> comparator) {
    //调用下面的private构造函数
    this(new TreeMap<>(comparator));
}

TreeSet(NavigableMap<E,Object> m) {
    this.m = m;
}

原理

其原理和HashSet一样，是依靠NavigableMap来实现排序的。

遍历方法

List

  1. for循环
List<String> list = new ArrayList<>();
for (int i = 0; i < list.size(); i++) {
    //遍历元素为list(i);
}
  1. foreach
List<String> list = new ArrayList<>();
for (String s : list) {
    //遍历元素为s
}
  1. 迭代器
List<String> list = new ArrayList<>();
Iterator iterator = list.iterator();
while (iterator.hasNext()){
    //遍历元素为iterator.next();
}

需要注意的是foreach方法是语法糖，编译后实际上还是使用的迭代器，因此在遍历的过程中不能对原集合进行增删操作。

错误示例

List<String> list = new ArrayList<>();
for (String s : list){
    if ("hello".equals(s))
        list.remove(s);
}
