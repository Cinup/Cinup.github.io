排序算法

发表于 2019-03-30 | 分类于 算法

简介

被面试官问到排序算法，回答得不是很好，整理一下。如果文字说明不清晰可以看动图演示。

冒泡排序

冒泡排序一种比较排序，它的思想很简单， 就是不断地比较相邻两个元素，大的向后移，就像泡泡一样向上冒，每一轮(并非一次)比较都能确定一个值的最终位置。

假设有如下的数组

(3，2，4，5，7，8，1)

第一轮排序:

3和2比较，3>2，交换3和2的位置

(2，3，4，5，7，8，1)

3和4比较，3<4，不用交换位置

4和5比较，4<5，不用交换位置

5和7比较，5<7，不用交换位置

7和8比较，7<8，不用交换位置

8和1比较，8>1，交换8和1的位置

第一轮冒泡排序结束

(2，3，4，5，7，1) [8]

然后继续下一轮的比较

其中(…)表示待排序元素，需要下一轮进行冒泡排序的的元素，[…]表示已排序元素，已经确定位置，不用再参与排序。

在时间复杂度上，冒泡排序需要两层循环，外层循环表示已排序的元素，内层循环选择最值。

因此n个元素冒泡排序的循环次数为(n-1)+(n-2)+…+1，最坏的时间复杂度为O(n²)，由于冒泡排序一轮排序是相邻元素依次比较，所以当没有发生元素交换时，说明元素已有序，只需要n-1次比较,最好的时间复杂度为O(n)。

选择排序

选择排序也是一种比较排序，它的思想是一轮比较找出最值，并把最值放到指定位置。它与冒泡排序不同的是，冒泡排序交换的很频繁，而选择排序一轮比较也只交换一次，就是将最值和指定位置上的元素交换。

假设有如下的数组

第一轮排序:

(3，4，2，8，5，7，1)

4和3比较，min=4

2和3比较，min=2

8和2比较，min=2

…

1和2比较，min=1

第一轮比较结束，3和1交换位置

[1] (4，2，8，5，7，3)

然后继续下一轮的比较

在时间复杂度上，选择排序也需要两层循环，外层寻遍表示最值放置的位置，内存循环表示待排序的个数。

因此n个元素选择排序的循环次数为(n-1)+(n-2)+…+1，最坏的时间复杂度为O(n²)，由于选择排序一轮排序都是和当前已知的最值比较，因此元素即使有序排序算法也不能感知到，最好的时间复杂度也是O(n²)。

插入排序

插入排序也是比较排序，插入排序的思想类似于整理扑克牌，抓一张牌，然后把牌放入合适的位置。冒泡排序和选择排序每轮排序都能确定一个元素的最终位置，但是插入排序是做不到的，因为它的新元素只和已知有序的序列比较。

假设有如下的数据

(3，4，2，8，5，7，1)

当只有一个数据时，是不需要进行插入排序的，因此实际插入排序是从第2个元素开始

[3] (4，2，8，5，7，1)

第一轮排序，要插入到元素是4

4和3比较，4>3，不用移动

[3，4] (2，8，5，7，1)

第二轮排序，要插入到元素是2

2和4比较，2<4，4向后移动

(3，4，4) (8，5，7，1)

2和3比较，2<3，3向后移动

(3，3，4) (8，5，7，1)

然后将2插入到当前位置(加粗的数字表示插入的位置)

[2，3，4] (8，5，7，1)

然后继续下一轮的比较

在时间复杂度上，插入排序也是两层循环，外层表示待排序的元素，内层选择插入的位置。

因此n个元素插入排序的循环次数为1+2+…+(n-1)，最坏的时间复杂度也是O(n²)，由于插入排序之前的结果是有序的，因此对于插入值，首先和最值比较，最好的情况是第一次就确定了插入的位置，无需再进行后续的比较，最好的时间复杂度为O(n)。

堆排序

堆排序的关键是构建堆，堆的定义类似完全二叉树。

假设有如下的数组

(3，4，2，8，5，7，1)

对于下标为k的元素，下标为2k+1和2k+2的元素就是它的孩子，如元素3的下标为0，那么下标为1(2*0+1)和2(2*0+2)的元素就是它的孩子。

堆分为大根堆和小根堆，大根堆要求父结点的值不小于孩子结点的值，所以大根堆中根结点的值就是最大值，小根堆则相反，父结点的值不大于孩子结点的值。

堆排序的第一步需要构建出堆，以大根堆为例，首先从最后一个非叶子结点到循环根结点，如果孩子结点中的较大值大于父结点的值，就交换它们的值，循环完成后，根结点的值就会是最大值。

然后再次从第二个结点(第一个结点为根结点，根结点已经是最大值)循环到最后一个非叶结点，确保父结点的值不小于孩子结点。

n个元素构建堆的时间复杂度为O(n)

堆建好后就可以进行排序了，堆排序的规则是，由于根结点是最大值，因此将根结点和最后一个结点交换位置，堆长度减一，表示确定了一个元素的位置，然后调整堆，使得根结点仍然是最大值。

在时间复杂度上，因为n个元素堆排序的循环次数为n-1次，调整次数为堆的高度，调整次数和为log(n-1)+log(n-2)+…+log2=log(n-1)!，所以时间复杂度近似为O(nlogn)。

快速排序

快速排序使用二分法的方式来进行排序，首先选择一个中间值，以这个中间值划分出左右两个区间，再对这两个区间再次进行划分。

假设有如下的数组

(3，4，2，8，5，7，1)

第一轮排序，选择3为中间值

4和3比较，4>3，不用交换

2和3比较，2<3，交换2和4

(3，2，4，8，5，7，1)

…

1和3比较，1<3，交换1和4

(3，2，1，8，5，7，4)

第一轮快速排序结束，交换3和1

(1，2) [3] (4，8，5，7)

对左边和右边两个区间继续排序

在时间复杂度上，快速排序近似于二分法。

因此n个元素快速排序循环次数最好情况为

T(n)=2T(n/2)+n-1=2(2T(n/4)+(n-1)/2+n-1=4T(n/4)+2(n-1)=…=2^kT(n/2^k)+k(n-1)

令n/2^k等于1，k=logn

T(n)=nT(1)+nlogn，因为T(1)=0，所以T(n)=(n-1)logn，所以时间复杂度为O(nlogn)

最差情况为

T(n)=T(n-1)+T(1)+n-1=T(n-2)+2T(1)+n-1+n-2=(n-1)+..+1，所以时间复杂度为O(n²)

归并排序

归并排序是采用分治思想的排序算法，即将先使子序列有序，再合并有序的子序列即可。

假设有如下的数组

(3，4，2，8，5，7，1)

以二路归并为例，先将数组分成两个序列

(3，4，2) (8，5，7，1)

再分别对两个子序列进行分组直到不能分组为止

{ { (3) } { (4) (2) } } { { (8) (5) } { (7) (1) } }

分组完成后进行排序

{ { (3) (2，4) } { (5，8) (1 ，7) } }

{ (2，3，4) (1，5，7，8) }

[1，2，3，4，5，7，8]

{…}表示进行需要排序的分组

在时间复杂度上，二路归并是二分法。

T(n)=2T(n/2)+n

所以归并排序的时间复杂度无论好坏一律为O(nlogn)。

计数排序

上面的排序都是比较排序，比较排序的时间复杂度下限为O(nlogn)，而计数排序是非比较排序，思想是用空间换时间，通过额外的空间来记录排序值来达到排序的目的，可以突破比较排序时间复杂度的下限。

但是计数排序的使用范围很有限，它对数据的要求比较严格，需要元素的大小局限一个小范围内，数据范围过大效果反而会不如比较排序。

假设有如下的数组

(91，91，98，95，95，97，93，99)

数据的范围在90~100之间，首先遍历一遍数组，找出最大值99和最小值91。确定额外的数组大小为10(99-91+1)，基础值为91，然后再遍历一遍数组，通过元素值减去基础值的差值作为元素在数组中的地址，然后将数组 地址对应的值加1。

最终额外数组中的值为

[2，0，1，0，2，0，1，1，1]

分别对应值为91～99的个数

最后遍历额外数组，输出其存储的对应值即可。

在时间复杂度上，计数排序的遍历次数为T(2n+k)，所以时间复杂度为O(max{n，k})，可见当k比较大，也就是数据的范围远大于数据的数量时，计数排序的时间复杂度为O(k)，当k>n²时，计数排序的效率还不如比较排序。

所以，计数排序的使用范围没有其他的排序方法那么广泛，它只适合数据量多但是数据范围比较集中(n>k)的整数，这时它的效率远高于比较排序，能以线性的时间复杂度完成排序。

总结

排序方法 时间复杂度 空间复杂度
冒泡排序 最差O(n²) 最好O(n) S(1)
选择排序 O(n²) S(1)
插入排序 最差O(n²) 最好O(n) S(1)
堆排序 O(nlogn) S(1)
快速排序 最差O(n²) 最好O(nlogn) S(logn)
归并排序 O(nlogn) S(n)
计数排序 O(max{n，k}) S(k)
