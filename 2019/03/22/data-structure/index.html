定义和概念

树是一种一对多的具有层次关系的数据结构，它在离散数学中定义非常抽象，但是实际表示出来又是十分直观的，直接看简单的例子:


上面的图片就是一颗简单的树，树其实也可以看作一种无环的图。

  • 结点

    树的每个元素都称作结点，结点与结点之间会存在连通的路径，如图中的1，2，3，4，5结点。

    • 根结点

    根结点是树开始的结点，如图中的1结点。

    • 孩子结点

    一个结点连通的后续结点叫做该结点的孩子结点，如图中的2，3，4都是1的孩子结点。

    • 父结点

    孩子结点的连通的前驱结点叫做父结点，如图中1结点是2结点的父结点。

    • 兄弟结点

    拥有相同的父结点的结点就是兄弟结点，如图中的2，3，4结点。

    • 叶子结点

    没有孩子结点的结点叫做叶子结点，如图中的3，4，5结点。

    • 分支结点

    既不是根结点，也不是叶子结点的结点叫做分支结点，如图中的2结点。

  • 结点的度

结点的孩子个数就是结点的度，1结点的度为3，2结点的度为1，3，4，5结点的度为0。

  • 树的度

所有结点的度的最大值就是树的度，图中树的度就是3。

  • 结点的高度

根结点的高度为1，其余结点的高度就是父结点的高度加1。

  • 树的高度

所有结点的高度的最大值就是树的高度。

  • 子树

树的概念是递归的，每个结点的子结点也可以看作以子结点为根的树。

  • 无序树

树的兄弟结点位置互换后与原来的树等价，这种树就称为无序树。

  • 有序树

树的兄弟结点位置互换后即是另一颗树，这种树就是有序树。

二叉树

定义

当树的结点至多只能有两个孩子结点时，这样的树就是二叉树。

二叉树遍历

因为二叉树一般是有序树，区分左右子树，因此会有不同的遍历方式，不同的遍历方式产生不同的结果。

以下图为例，来说明图的遍历方式。

  • 先序遍历
当前结点—>左结点—>右结点

遍历结果为:1—>(2—>4—>5)左子树—>(3—>6)右子树

先序遍历第一个元素就是根元素。

  • 中序遍历
左结点—>当前结点—>右结点

遍历结果为:(4—>2—>5)左子树—>1—>(3—>6)右子树

中序遍历以根元素为界划分左右子树。

  • 后序遍历
左结点—>右结点—>当前结点

遍历结果为:(4—>5—>2)左子树—>(6—>3)右子树—>1

后续遍历最后一个元素就是根元素。

  • 层序遍历
从上倒下—>从左到右

遍历结果为:1—>2—>3—>4—>5—>6

从遍历结果可知，除了层序遍历，树的遍历逻辑上就是遍历左右子树，是递归的遍历，因此代码很自然的就可以采用递归的方式遍历。

满二叉树

除了最后一层是叶子结点外，每一层上的所有结点都有两个子结点。

完全二叉树

若将一颗树从上到下，从左到右的次序编号，若一颗树的编号都是连续的，那么这颗树就是完全二叉树。满二叉树也是一颗完全二叉树，完全二叉树就是满二叉树少掉最后一层的右边部分的叶子结点。

二叉排序树

二叉排序树是一颗特殊的二叉树，它要求每个结点:

  • 结点的左孩子为空或者值小于该结点的值
  • 结点的右孩子为空或者值大于该结点的值

因此，暂时先不考虑相等的情况(相等的情况可以自由分配到左或者右)，二叉排序树的所有结点，左子树上的值都小于它的值，右子树上的值都大于它的值，二叉树排序树中序遍历就可以得到一个顺序的结果。

二叉排序树的查找

二叉排序树的查找效率非常好，和二分查找一样。

  • 若查找值等于当前结点的值，返回找到；
  • 若查找值小于当前结点的值，从当前结点的左子树中查找；
  • 若查找值大于当前结点的值，从当前结点的右子树中查找；
  • 若当前结点为空，返回未找到。

二叉排序树的插入

二叉排序树的插入过程(相等情况自由选择插入左还是右):

  • 若当前结点为空，则插入当前结点；
  • 若插入值小于当前结点的值，则插入当前结点的左子树中；
  • 若插入值大于当前结点的值，则插入当前结点的右子树中。

二叉排序树的删除

二叉排序树的删除要考虑结点的类型:

  • 若为叶子结点，直接删除；
  • 若为只有一个孩子的分支结点，直接用孩子结点代替当前结点；
  • 若为有两个孩子的分支结点，从右子树中找到前驱(或者后继)代替当前结点。

关于上面的第三条，解释一下前驱和后继，所谓前驱就是这个二叉排序树中序遍历时排在它前面的结点，后继就是排在后面的值。在树中前驱的表现就是左子树中的最大值，后继的表现就是右子树中的最小值。

平衡二叉树

二叉排序树虽然在查找上表现优异，但是也还存在着缺点。在树构建时，若插入的值基本有序，那么二叉排序树会 退化成链表，这时查找的时间复杂度会变成最差的情况，原因就是树的左右子树不平衡，由此在二叉排序树的基础上加入了平衡的条件。

平衡二叉树是一颗自平衡的二叉排序树，它要求每个结点左右子树的高度差不超过1。

失衡

在二叉树的插入和删除过程中可能会造成二叉树的失衡，失衡的情况有4种:

  • LL型:在8结点的左子树的左子树添加2结点导致失衡
  • LR型:在8结点的左子树的右子树添加6结点导致失衡
  • RL型:在5结点的右子树的左子树添加6结点导致失衡
  • RR型:在5结点的右子树的右子树添加10结点导致失衡

旋转

在上面4种失衡的情况中，LL和RR是对称的，LR和RL也是对称的。

因此处理失衡的方法就需要2种:LL型和RR型的处理方法，LR型和RL型的处理方法。

对于LL型或者RR型失衡的二叉树，只需要一次旋转，将失衡点向失衡的方向(高度低的方向)旋转即可。

红黑树

红黑树是一种特殊的平衡二叉树，它的平衡条件不是左右子树的高度，它的结点额外带有颜色属性且必须满足条件(平衡条件):

  • 红黑树的结点为黑色或者红色
  • 根结点必须是黑色
  • 红色结点的孩子结点必须是黑色
  • 每个叶子结点(空结点)是黑色的，这个叶子结点并不是真正的叶子结点，而是真正叶子结点下一个黑色NULL结点，只有颜色，并没有值
  • 任意结点到每个叶子结点路径上的黑色结点数目都相等

所以红黑树是根据路径上的黑色结点数量来保持平衡的，因此插入一个结点时，最好将结点的颜色默认为红色，这样可以避免破坏第五条规则。

添加

红黑树的插入同样是根据二叉树排序树的插入方式，由于插入的结点默认为红色，当且仅当插入结点的父结点为红色时，红黑树的平衡条件(红色结点的孩子结点必须是黑色)被打破。

删除

由于添加结点指定了颜色为红色，所以添加只会破坏红色结点的孩子不能为红色这一平衡条件。

但是删除结点就不一样了，删除的结点可能为红色，也可能为黑色。在这过程中可能会破坏多个条件。

首先删除结点过程和二叉树的删除过程是一样的，在删除之后，需要根据删除结点的位置以及颜色对红黑树进行重新着色。

在上面的二叉树中，删除的结点有三种情况，但是对于有两个孩子结点的情况，会用后继结点来代替这个结点，其实可以理解为删除的是那个后继结点，而后继结点必然是叶子结点或者只有一个孩子的结点，因此删除情况可以归结为两种。

  • 叶子结点
  • 只有一个孩子的结点

由于红黑树的特性，只有一个孩子结点不存在以下的情况:

  • 黑色结点只有一个包含黑色结点的孩子(右孩子情况是一样的)

  • 红色结点只有一个包含黑色结点的孩子

两个孩子结点=叶子结点+只有一个孩孑结点，由于上面的分析不存在只有一个子树的红色结点(红色不允许违法条件4，黑色违反条件5)。
