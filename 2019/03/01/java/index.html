区别

  • java中 == 和equals 和hashCode的区别

基本数据类型:byte,short,char,int,long,float,double,boolean比较时,只能使用==,比较的是它们的值。引用数据类型:使用==比较时,比较的它们的内存地址,即它们是否指向同一个对象。

equals是Object类的一个方法,默认实现是内存地址的比较,即引用数据类型的==比较,子类可根据具体情况需要重

写该方法。

hashCode也是Object类的一个方法,它返回一个int类型的整数,这个整数的作用是确定该对象在哈希表中的索引位置。

Object的hashCode方法的计算依赖于对象实例的内存地址,故每个Object对象的hashCode都是唯一的。

所以在没有重写hashCode的情况下,若两个对象 equals为true,则hashCode一定相等, 但是hashCode相等,

equals不一定为true。

因此在集合中,判断两个对象是否相等:

  1. 先比较hashCode,hashCode不相等,则对象一定不相等。
  2. hashCode相等再判断equals。

  • final finally和finalize的区别

    1. final是修饰符，可以修饰变量，方法和类，被修饰的变量赋值后就不能被修改，被修饰的方法不能被重写，被修饰符类不能被继承。
    2. finally是异常处理的一部分，配合try/catch使用，附带一个语句块，在try/catch中无论是否发生异常都被执行，通常用来释放资源，例如关闭打开的文件。
    3. finalize是Object的方法
这个方法在gc时被调用，但是调用后不意味着对象立即被回收，在java9后被弃用。

  • 序列化的方式

序列化:就是把java对象转化为字节序列的过程，反之称为反序列化。

对象的序列化主要有两种用途

  1. 对象的持久化
  2. 在网络上传输对象字节序列

实现序列化的方式

  1. 实现Serializable接口

一个对象想要被序列化，那么它的类就要实现此接口，这个对象的所有属性(包括private属性、包括其引用的对象)都可以被序列化和反序列化来保存、传递。但是transient和static修饰的变量不会被序列化。

transient:它声明的变量实行序列化操作的时候不会写入到序列化文件中去。例如ArrayList源码中

serialVersionUID:字面意思上是序列化的版本号，凡是实现Serializable接口的类都有一个表示序列化版本标识符的静态变量，可以用来区分类的版本。

  1. 实现Serializable接口，并添加writeObject方法和readObject方法
  2. 实现Externalizable接口

Serializable接口的子类，有时我们不希望序列化那么多，可以使用这个接口，这个接口的writeExternal()和readExternal()方法可以指定序列化哪些属性。

##理解

  • 抽象类的意义
  1. 为子类提供一个公共的类型;
  2. 封装子类中重复内容(成员变量和方法);
  3. 定义有抽象方法,子类虽然有不同的实现，但该方法的定义是一致的。
  • 多态的理解

多态就是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编译时并不确定，而是在程序运行期间才确定，即一个引用变量倒底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在由程序运行期间才能决定。因为在程序运行时才确定具体的类，这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

  • 能否创建一个包含可变变量的不可变对象

不可变类:所谓的不可变类是指这个类的实例一旦创建完成后，就不能改变其成员变量值。

所以一个不可变类需要满足以下条件

  1. final修饰，保证其不会继承
  2. 所有成员变量使用private和final修饰
  3. 只提供getter方法，不提供setter方法
  4. 提供带参数的构造方法，方法内部要用深拷贝对成员变量赋值

因此如果要构造一个包含可变变量的不可变对象，在理论上是可以的，只要满足上述条件保证成员变量不被共享就能使得成员变量无法被修改。

  • 静态属性和方法是否可以被继承和重写，原因是什么

可以被继承，但是不能被重写，而是被隐藏。静态属性和方法是属于类的，是所有对象公用的，子类可以继承父类的静态方法，但是不同通过父类引用指向子类对象实现静态方法的多态，因此说可以被继承而不能被重写。

  • 成员内部类，静态内部类，局部内部类和匿名内部类

内部类:可以将一个类定义在另一个类或者方法里面，这样的类称之为内部类。

  1. 成员内部类

成员内部类是最普通的内部类，相当于外部类的一个属性。

  • 成员内部类可以使用权限修饰符修饰。
  • 成员内部类可以无条件访问外部类的所有属性和方法。
  • 不能在成员内部类内定义静态的属性，方法。

成员内部类使用组合间接实现多继承。

  1. 静态内部类

使用static修饰的成员内部类就是静态内部类。

  • 静态内部类和静态属性方法一样，属于类而不属于对象，所以静态内部类只能访问外部静态属性和方法。
  • 外部类不能访问内部类的非静态属性和方法。
  1. 局部内部类

定义在代码块或者方法体内部的类就是局部内部类。

  • 不能使用权限修饰符，因为不属于类的属性。
  • 对外部完全隐藏，只能在其作用域内生成对象。
  • 局部内部类访问作用域内的局部变量，该局部变量需要使用final修饰。
  1. 匿名内部类

匿名内部类是没有名字的内部类，没有办法引用它，直接使用new创建实例。

  • 匿名内部类必须继承一个类或者实现一个接口。
  • 可以在匿名内部类定义代码块，用于实例化，但是不能定义静态代码块。
